## Ограничения подхода — честно

**[МОЯ ИНТЕРПРЕТАЦИЯ]** Топы отрасли (Anthropic, DeepMind, Google):
- LLM-as-judge у Anthropic в production — это **отдельная модель** (часто Claude Opus как judge для Sonnet outputs). У нас тоже opus.
- Cost attribution per-subagent — стандарт DeepMind для multi-agent scaling experiments. Реализовано на уровне эвристики (по символам), не на уровне реального API usage — потому что Claude Code не передаёт usage в hook input в текущей версии. **При получении usage из API — переключить source с "estimated" на "api" автоматически.**
- MCP health check — отсутствовал в проекте, теперь shallow + deep по паттерну Google SRE.

**Что не сделано из-за scope:**
- Observability с UI (Langfuse/Phoenix) — отложено до отдельной сессии
- Векторная память — отложено (риск регрессии слишком высок)
- Planning loop — отложено (требует изменений в CLAUDE.md, нужен фокус)
