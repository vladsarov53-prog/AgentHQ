# Золотые тест-кейсы: кросс-агентные сценарии

Используются при обновлении маршрутизации или промптов для проверки мультиагентных задач.

---

## Тест 1: Отчёт ФСИ (accounting + operations параллельно)

**Вход:** «Подготовь данные для промежуточного отчёта по гранту ФСИ.»

**Ожидаемое поведение:**
- Диспетчер определяет кросс-ролевую задачу (accounting + operations)
- Параллельно: accounting (расходы по смете, план/факт) + operations (статус вех, дедлайны)
- Последовательно: диспетчер собирает результаты в единый документ
- Каждый субагент указывает конкретные источники (файлы, записи в памяти)

**Красные флаги:**
- Один субагент вместо двух (неполное покрытие)
- Последовательный вызов вместо параллельного (потеря эффективности)
- Расхождение данных между субагентами (сумма accounting != сумма operations)
- Отсутствие сборки результатов диспетчером

## Тест 2: Подготовка к встрече с контрагентом (operations + legal параллельно)

**Вход:** «Подготовь к созвону с CAD-разработчиком Алексеем.»

**Ожидаемое поведение:**
- Диспетчер определяет кросс-ролевую задачу (operations + legal)
- Параллельно: operations (статус задач, открытые вопросы) + legal (обязательства по договору, дедлайны)
- Последовательно: диспетчер формирует повестку встречи
- Повестка: 3-5 пунктов с источниками

**Красные флаги:**
- Только статусы без юридического контекста (или наоборот)
- Обязательства из договора не привязаны к конкретному файлу и пунктам
- Повестка без конкретных вопросов для обсуждения
- Пропущены финансовые аспекты (оплата этапов)
