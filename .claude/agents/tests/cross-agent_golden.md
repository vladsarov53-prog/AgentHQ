# Золотые тест-кейсы: кросс-агентные сценарии

Используются при обновлении маршрутизации или промптов для проверки мультиагентных задач.

---

## Тест 1: Отчёт ФСИ (accounting + operations параллельно)

**Вход:** «Подготовь данные для промежуточного отчёта по гранту ФСИ.»

**Ожидаемое поведение:**
- Диспетчер определяет кросс-ролевую задачу (accounting + operations)
- Параллельно: accounting (расходы по смете, план/факт) + operations (статус вех, дедлайны)
- Последовательно: диспетчер собирает результаты в единый документ
- Каждый субагент указывает конкретные источники (файлы, записи в памяти)

**Красные флаги:**
- Один субагент вместо двух (неполное покрытие)
- Последовательный вызов вместо параллельного (потеря эффективности)
- Расхождение данных между субагентами (сумма accounting != сумма operations)
- Отсутствие сборки результатов диспетчером

## Тест 2: Подготовка к встрече с контрагентом (operations + legal параллельно)

**Вход:** «Подготовь к созвону с CAD-разработчиком Алексеем.»

**Ожидаемое поведение:**
- Диспетчер определяет кросс-ролевую задачу (operations + legal)
- Параллельно: operations (статус задач, открытые вопросы) + legal (обязательства по договору, дедлайны)
- Последовательно: диспетчер формирует повестку встречи
- Повестка: 3-5 пунктов с источниками

**Красные флаги:**
- Только статусы без юридического контекста (или наоборот)
- Обязательства из договора не привязаны к конкретному файлу и пунктам
- Повестка без конкретных вопросов для обсуждения
- Пропущены финансовые аспекты (оплата этапов)

## Тест 3: Недельный брифинг (operations + accounting -> strategy)

**Вход:** «Понедельничный стратегический брифинг.»

**Ожидаемое поведение:**
- Диспетчер определяет смешанную задачу (ops + acc -> strategy)
- Параллельно: operations (статусы, задачи) + accounting (burn rate, расходы)
- Последовательно: strategy (анализ поверх данных обоих агентов)
- Strategy НЕ дублирует ops/acc, а даёт стратегический слой

**Красные флаги:**
- Только один субагент (неполное покрытие)
- Strategy пересказывает данные вместо анализа
- Нет burn rate данных от accounting
- Нет уровней уверенности в рекомендациях strategy

## Тест 4: Pitch-deck с финансами (strategy + accounting)

**Вход:** «Подготовь контент для питч-дека инвестору с финансовыми данными.»

**Ожидаемое поведение:**
- Диспетчер определяет параллельную задачу
- Параллельно: strategy (позиционирование, USP, рынок) + accounting (финансы, burn rate)
- Последовательно: сборка контента для слайдов
- Каждый слайд = одна мысль, цифры крупно

**Красные флаги:**
- Финансовые данные без accounting-agent
- Позиционирование без strategy-agent
- Абстрактные тезисы без привязки к данным проекта

---

## Тест 5: Договор с бюджетом (legal + accounting)

**Вход:** «Проанализируй новый договор и проверь совместимость с бюджетом гранта.»

**Ожидаемое поведение:**
- Параллельно: legal (обязательства, пункты) + accounting (бюджет, остаток)
- Последовательно: диспетчер сверяет и даёт рекомендацию
- Рекомендация учитывает оба аспекта (юрид. + финансовый)

**Красные флаги:**
- Только юр. анализ без бюджетной проверки (или наоборот)
- Сумма договора не сверена с бюджетом
- Рекомендация игнорирует один из аспектов
