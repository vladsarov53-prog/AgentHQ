# Frontend Design

Создание продуктовых веб-интерфейсов с высоким качеством дизайна. Не generic AI-слоп, а осмысленный, запоминающийся дизайн.

## Процесс

### 1. Дизайн-мышление (до кода)

Перед написанием кода определи:
- **Назначение**: какую проблему решает, для кого
- **Тон**: выбери эстетику (минимализм, бруталист, ретро-футуризм, luxury, playful, и т.д.)
- **Ограничения**: технические требования
- **Отличие**: что делает интерфейс запоминающимся

### 2. Пять столпов эстетики

**Типографика**: характерные шрифты, не Inter/Arial/Roboto по умолчанию. Google Fonts дают огромный выбор.

**Цвет и тема**: связная палитра. Доминантный цвет + акценты. Не робкие пастели, а смелые решения.

**Анимации**: осмысленные, не ради галочки. CSS transitions, staggered reveals, микроанимации при взаимодействии.

**Композиция**: неожиданные раскладки, асимметрия, overlap, generous negative space. Не всё по центру.

**Фоны и детали**: gradient mesh, текстуры, паттерны, многослойность, тени с характером.

### 3. Антипаттерны (избегать)

- Purple gradient на белом фоне
- Всё по центру, одинаковые карточки
- Inter + rounded corners + тонкие линии
- Одинаковый шаблон для разных проектов
- Предсказуемый hero-section -> features -> CTA

### 4. Технический стек

Выбирай в зависимости от задачи:

| Задача | Стек |
|---|---|
| Быстрый прототип | Vanilla HTML + CSS + JS |
| Дашборд | React + Tailwind + shadcn/ui |
| Лендинг | HTML + CSS (или Next.js) |
| Компонент | React + TypeScript |
| Демо/артефакт | Single HTML file |

### 5. Принципы

- Intentionality, not intensity. Каждое решение осмысленно.
- Mobile-first, если не указано иное.
- Accessibility: контрастность, клавиатурная навигация, aria-labels.
- Performance: lazy loading, optimized images, минимум JS.

## Финальная проверка (обязательна)

ПЕРЕД заявлением о готовности UI-изменения:

1. Запустить приложение / dev-сервер
2. Сделать скриншот через `webapp-testing` (Playwright `page.screenshot()`) или `mcp__Claude_in_Chrome__computer`
3. Сверить с требованиями: расположение, цвет, текст, размеры, состояния (hover/active/disabled), отзывчивость
4. Если скриншот технически невозможен (headless среда без рендера, нет dev-сервера) — добавить маркер «⚠️ визуально не проверил, причина: ...» в финальный ответ

Без выполнения этого шага задача считается невыполненной. Hook `screenshot-gate.py` (Stop) блокирует завершение при отсутствии скриншота или маркера.

**Why:** calibration_log 2026-05-04 — HUD-панель VoiceTypingRU: код выглядел корректно, но визуально результат не соответствовал требованиям (расположение, цвет, поведение волны). Проверка через код не ловит реальные визуальные дефекты.
